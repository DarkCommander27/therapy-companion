### Short project description

CareBridge Companion is an open source, privacy-first check-in companion designed for youth in residential care settings. It supports empathetic youth check-ins, staff briefings, and safeguarding alerts while keeping deployment and data control local to the facility whenever possible.

### Tell us about the project

CareBridge Companion is a pilot-stage open source software project focused on supportive, non-clinical check-ins for youth in residential care environments. The system is designed to help facilities support youth through empathetic conversation, surface potential safety concerns to staff through alerts and briefings, and preserve privacy through local-first deployment patterns and optional local model inference.

The project is intentionally scoped with clear boundaries. It is not therapy, not diagnosis, not a crisis hotline, and not a staff-to-youth messaging system. The repository currently includes an application prototype, security hardening work, safeguarding documentation, contributor health files, and a scripted demo path for evaluators.

### Tell us who you are and why you are interested in fiscal hosting with OSC

I am the current maintainer of CareBridge Companion and the person responsible for the public repository and project direction at this stage. I am interested in fiscal hosting because I want the project to have a transparent, accountable structure for receiving and managing support as it matures beyond a single-maintainer pilot. If the project continues to develop, fiscal hosting would help cover the administrative and legal overhead of handling funds so more attention can stay on product safety, documentation, contributor onboarding, and open source maintenance.

OSC seems aligned with the kind of project I am trying to build: open source software with a public repository, community-facing documentation, and a need for a more durable financial and governance structure than an individual maintainer account can provide.

### Why is this project open source?

This project is open source because trust, reviewability, and reuse matter in this domain. Facilities and contributors should be able to inspect the code, understand the system boundaries, assess the privacy and safeguarding posture, and adapt the software to their own operational needs. An open repository also makes it easier to improve security, documentation, governance, and deployment practices in public rather than treating sensitive operational choices as an opaque black box.

### What problem does the project solve?

The project is aimed at a gap in residential care settings where youth may need a supportive way to check in outside scheduled conversations, while staff still need consistent safety visibility and accountability. CareBridge Companion is designed to help with supportive check-ins, flag possible safety concerns, and generate staff-facing context without claiming to replace therapists, crisis services, or direct human care.

### What stage is the project in today?

The project is currently a pilot-stage working prototype in active testing and documentation hardening. It is public, under active development, and includes substantial documentation, security-oriented implementation work, and automated tests, but it should not be described as production-certified.
